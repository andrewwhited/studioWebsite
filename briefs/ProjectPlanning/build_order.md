# Andrew Whited — Project Stages

## Working Note

This document tracks the overall stage of the project and the remaining phases required to complete the site.

The original build order assumed the site was starting from zero. That is no longer the case. The site now exists in an early functional state.

The goal now is to move through the remaining stages in a deliberate way so that the visual system, editorial structure, and commerce layer develop coherently.

This document describes **project stages**, not detailed implementation tasks.

---

# Current Project State

The site currently exists in an early working form.

The following foundations are already established:

- core site structure
- initial layouts
- working navigation
- base templates
- chosen technology stack

The remaining work focuses on **refining the system, populating content, integrating commerce, and polishing the experience**.

---

# Stage 1 — Structural Foundation

### Purpose

Establish the core site structure and confirm that the IA works in the browser.

### Outcome

The site has a working framework with the primary sections present.

This stage is largely complete.

---

# Stage 2 — Visual System Refinement

### Purpose

Clarify and refine the visual logic that governs the site.

This includes the relationship between typography, spacing, layout rhythm, and imagery.

### Focus

- typography hierarchy
- spacing rhythm
- grid logic
- layout pacing
- image treatment
- section distinction

### Outcome

The site should begin to feel like **one coherent visual system** rather than a set of page templates.

This is the **current primary stage** of the project.

---

# Stage 3 — Template Maturity

### Purpose

Confirm that the core page types function well when composed with real or realistic content.

### Focus

The goal is to ensure that the main page types feel stable and convincing as part of a larger editorial system.

### Outcome

The site should demonstrate that its layout logic works consistently across its main sections.

---

# Stage 4 — Editorial Structure

### Purpose

Organize the content layer that supports the editorial side of the site.

This includes the structure for collections, works, photo sets, and studio content.

### Approach

Begin with a file-based structure rather than introducing a CMS immediately.

### Outcome

The site gains a manageable content structure that can evolve as the project grows.

---

# Stage 5 — Content Population

### Purpose

Populate the site with real material once the templates and system feel stable.

### Outcome

The site transitions from a structural prototype into a meaningful editorial presentation.

Content entry at this stage may still drive minor layout adjustments.

---

# Stage 6 — Commerce Layer

### Purpose

Add the store and transaction functionality once the editorial experience is working well.

### Principle

The store should inherit the site's visual system rather than defining it.

### Outcome

The commerce layer integrates cleanly with the rest of the site rather than becoming the dominant experience.

---

# Stage 7 — UX Portfolio Subdomain

### Purpose

Add the UX portfolio as a related but separate site once the primary system is stable.

### Outcome

The UX work exists as its own environment while still feeling connected to the broader studio presence.

---

# Stage 8 — Refinement and Testing

### Purpose

Tighten the experience and confirm the system holds together under real conditions.

### Focus

- responsive refinement
- visual consistency
- interaction clarity
- content coherence
- performance and optimization

### Outcome

The site moves from functional to convincing.

---

# Stage 9 — Launch

### Purpose

Publish a coherent first version of the site.

Launch should happen once the system feels intentional and stable rather than waiting for perfect completeness.

---

# Guiding Principle

Build the experience first.

Then stabilize the editorial layer.

Then add commerce.

Then refine and polish.

Avoid letting technical implementation or store mechanics determine the character of the site.
