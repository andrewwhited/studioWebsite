01

Overview

The foundations are correct. The ambition is genuine. The system mostly holds. What it is not yet is finished — and several current weaknesses will resolve with real content and a final pass. The ones that won't self-resolve are what this report is for.

This is a technically disciplined site built around a coherent editorial philosophy. Every page uses the same 12-column grid at the same gutter and gap values. The navigation participates in the grid at the same column widths as every content section. The three-color text hierarchy (text / muted / rule) is applied consistently throughout. The token system is comprehensive and mostly honored. The section differentiation — each page having a distinct character without becoming disconnected from the whole — is more difficult than it looks and the implementation earns it.

Where the site falls short: there are too many distinct type styles (13 is too many; 8 or 9 is the target). Some clamp() ranges don't land on type scale steps. A handful of spacing values violate the 4px base unit. Some viewport-relative values create positional unpredictability. And the site is compositionally conservative — correct and clean, but not yet bold. The layouts are technically right but don't yet make a strong visual argument in the way the best Swiss and editorial work does.

The biggest unresolved risk is not code — it is content. The site is designed around photography that does not yet exist. Until it does, no final judgment about atmospheric success is possible. The placeholder panels suggest the right scale and proportion relationships. But the editorial warmth the brief describes has no vehicle yet.

Methodology

Four frameworks applied simultaneously. No one frame is sufficient alone. Together they give a complete picture.

Heuristic Evaluation

Jakob Nielsen's 10 usability heuristics applied across all page templates. Most operative for this site: Consistency and standards — are spacing, type sizes, and interactive behaviors consistent? Aesthetic and minimalist design — every non-load-bearing element is a liability. Visibility of system status — is location always clear?

Swiss Design Principles

Crouwel: grid as liberation, not constraint. Application: is the grid doing structural work, or just constraining placement? Hofmann: point, line, plane — visual contrast as structural grammar. Evaluate scale drama and figure/ground clarity. Müller-Brockmann: "The grid system is an aid, not a guarantee." The designer's judgment fills the space the grid enables.

Elements of Typographic Style

Bringhurst's key principles applied: typography exists to honor content; the measure (45–75 characters per line); hierarchy through scale and weight, not color; the page is a proportion — margin and column mathematics must be deliberate, not arbitrary.

Intention Alignment

The brief package is treated as the primary evaluative standard. The question is not only "is this well-executed?" but "is this the right problem to be solving?" Design thinking: evaluate fitness of solution before quality of execution.

"I am a functionalist troubled by aesthetics." — Wim Crouwel. The self-description that should sit above this entire project.
03

Grid & Layout

The grid discipline is the strongest element of the site. Every page template uses the same system, and the column assignments are not just consistent — they are meaningful.

The 12-column grid with 24px column gap and 48px outer gutter is the structural backbone. Every section — including the navigation — uses repeat(12, 1fr) with identical padding. This creates continuous vertical key lines from top to bottom of every page: the wordmark, the nav links, the left content edge, the right rail, all share the same x-positions across all pages. This is Crouwel's principle in practice.

The consistent column assignment pattern — label/wordmark in cols 1–2, primary content in cols 3–9, right rail in cols 9–12 or 10–12 — mirrors the nav's own structure. When the reader sees nav links and page content aligning to the same column, the system becomes legible. This is correct editorial thinking.

One section — perhaps the Collection hero, or a future editorial spread — could use the grid more dramatically: full-bleed type at scale, an unexpected ratio, asymmetric content placement. The system is set up for this. The opportunity hasn't been taken yet.

The Scale

The minor third (1.2) from a 14px base, with 4px-aligned fixed line heights, is correct and well-implemented in globals.css. The progression from 11.7px (xs) through 41.8px (H1) gives seven usable steps. Line heights from 16px to 48px are clean multiples that create genuine vertical rhythm. This is the right foundation.

Weight Convention

The three-weight system is applied correctly throughout most of the site: Light (300) for display and titles, Regular (400) for body text, Medium (500) for labels, tags, and metadata. This is the correct Swiss convention — weight does semantic work. One violation: the service card titles in Studio use Medium weight while functioning as display titles, not labels. This breaks the convention without justification.

Type Style Inventory

Across all 14 modules, the following distinct type treatments exist. Thirteen is too many. Architecture has few materials, used with precision. The target is 8–9 styles maximum.

The clamp() Problem

Six clamp() uses across the site set fluid type sizes. Four of these set min or max values that don't land on type scale steps. These ranges should be re-anchored to scale steps.

Violation
Home title: max value not on scale
clamp(20px, 2.5vw, 36px) — 36px is the line-height of H3, not a font size. H2 is 34.8px, H1 is 41.8px. The title of the site, the primary identity marker, currently sits between scale steps at its maximum. Consider anchoring to H1 (2.985rem) as minimum.
home.module.css → .title
Violation
Collection hero title: both endpoints off-scale
clamp(32px, 5vw, 64px) — neither 32px nor 64px is in the type scale. This is the first text visitors see on a collection page, at full-bleed scale. Should anchor to H1 (41.8px) to display (4.5rem = 63px).
collection.module.css → .heroTitle
Warning
Objects entry title and Collection intro: off-scale minimums
clamp(18px, 2vw, 24px) and clamp(18px, 2vw, 26px) — 18px appears in both as a minimum, but 18px is between body (14px) and H6 (16.8px). Use 16.8px (H6) as minimum and 24px (H4) as maximum.
objects.module.css → .entryTitle / collection.module.css → .intro
Opportunity
The home page name should be more committed
At a maximum of 36px in font-weight light, the site name reads cautiously. The Chipperfield, Jasper Morrison, and Caruso St John sites all lead with the name at a commanding size. The light weight is correct. The size should be larger — anchored to H1 (2.985rem) minimum.

Color & Interaction

The color system is the strongest component of the site. Five values, rigorously applied. The warmth of the cream background creates the right atmospheric register without falling into lifestyle-brand territory.

The palette: background (#F6F6F2 cream), text (#111110 near-black), muted (#72726E gray), rule (#DEDED8 light border), placeholder (#3E3C38 dark). This is nearly monochromatic — the Swiss design principle applied to a warmer register. The cream versus near-black reads as material and warm without softening the structure.

Palette Size
5 values
Minimum viable and sufficient. No color introduced for decoration. Müller-Brockmann would approve.
Hover Model
Opacity reduction
Consistent: 0.5 for nav/labels, 0.6–0.7 for cards. 0.15s transition throughout.

Page Analysis

Each page evaluated for design system adherence, compositional strength, brief alignment, and outstanding issues.

Home / A−

The clearest expression of the grid as composition. Image (cols 1–4), void (cols 5–8), text (cols 9–12). The void is active, not empty. The column assignment mirrors the nav, creating a top-to-bottom alignment at the right third of the page. This is correct Hofmann thinking.

Opportunity
The name is undersized for its purpose
The home page has one job: establish identity. Andrew Whited at clamp(20px, 2.5vw, 36px) in font-weight light is cautious for a name that carries the entire site. At 1440px this renders at approximately 36px — the H3 line-height, not the H1 size. The Chipperfield, Jasper Morrison, and Caruso St John sites all lead with the name at a committed, commanding size. The light weight is correct. The size should be larger.

"Studio, objects, art, and image." at --text-xs (11.7px) with max-width: 160px. At 11.7px the line length will be approximately 20–25 characters — near the lower bound of Bringhurst's acceptable range (45–75). Consider sm (14px) with wider max-width.
Strength
The void is the design
Cols 5–8 are intentionally empty and the code comment says so. This is a rare and correct decision. The void creates breath, forces the eye to find the text, and creates compositional tension. This should not be changed.

The "Required Reading" section — 12 items (books, films, essays) in a tabular list with floating hover thumbnails — establishes cultural voice without text-heavy biography. Objects /objects A−

The split-panel layout with a sticky left panel and scrollable entry list is correctly conceived for an editorial discovery layer. The hover background image effect is the site's most distinctive interaction.

Strength
Hover background images — the site's most distinctive interaction
Clicking/hovering a collection row fades in a masked background image of the collection — centered, softened at edges with a linear gradient mask. The sticky left panel uses padding-top: 38vh to float the title text in the lower third. At a 500px viewport (some tablets in landscape) this is 190px; at 1200px it is 456px. Consider a fixed value (--sp-20: 160px) that achieves similar spatial effect without viewport-dependency.

Entry typography could be stronger
The entry title at clamp(18px, 2vw, 24px) is conservative. For a list that represents the primary editorial content of the Objects section, these titles could carry more weight in size. At 24px maximum they are H4 level. Consider H3 (29px) or H2 (35px) as maximum.
Art /art A−

The horizontal scroll works strip is the most formally ambitious layout on the site. The 50/50 viewport split, image cards at 50svh height, and the scrollable gallery format treat the browser as a display case. This is the right approach for artwork.

Strength
Horizontal scroll as format-conscious design
Card widths derived from calc(50svh * 6/7) maintain the 6:7 aspect ratio regardless of viewport height. Images are exactly 50svh — pinned to the viewport midpoint. This is not a workaround for overflow; it's a conscious treatment of the page as a horizontal gallery.
Opportunity
The stage void needs to earn its space
The top half of the Art page is entirely empty except for a small label and intro text. With placeholder content it reads as blank. With a strong artwork image or a compelling typographic moment, it would read as powerful. The design is set up for something; it doesn't yet deliver it.

Image /image A−

Clean and correctly designed. The 4-col grid with 2px gaps creates a precise contact-sheet quality appropriate for photography. The lightbox is well-engineered. The header correctly mirrors the Art page's right-pinned label convention.

09

Intention vs. Execution

The brief is unusually good. It is specific, self-aware, and anti-generic. Most of it is being honored. A few principles are being honored in structure but not yet in practice.

What the Brief Says — What the Site Does

Strength
"Editorial over corporate" — Being honored
The layouts do not read as templates. The Objects hover effect, the Art horizontal scroll, the Collection hero — these feel authored. The site does not look like Squarespace.
Strength
"Restraint over decoration" — Being honored
There is one exception (corner-mark CTA) but the site is otherwise clean. No gradients, no shadows, no illustration, no icon libraries. The restraint is genuine.
Strength
"Distinct modes within one system" — Being honored
Home (hero split), Studio (long-form), Objects (list), Art (scroll gallery), Image (grid), Store (filter+grid) — each is visually distinct while sharing the same type, spacing, color, and grid vocabulary. This is the site's strongest achievement.
Warning
"Typography as architecture" — Partially honored
The grid uses type correctly as structure. But with 13 type styles, the typographic architecture is not as clear as it should be. Architecture has few materials, used with precision and intention. When the type system is tightened to 8–9 styles, the remaining styles will carry more weight and meaning.
Warning
"Image-led experience" — Structurally honored, atmospherically pending
Every layout prioritizes image in scale and placement. The bleeds, the horizontal scroll, the 70svh collection hero — these are image-led decisions. But there are no images. The atmosphere the brief describes cannot be assessed until photography exists.
Warning
"Negative space should be active and meaningful" — Mostly honored
The home void is active. The Art stage void is motivated. But the Objects left panel's 38vh push creates negative space above the text that feels positional rather than compositional. Active negative space creates tension; positional emptiness is just a gap.
Warning
"Commerce without dilution" — Structurally correct, functionally incomplete
The Store doesn't feel like Amazon. The editorial hierarchy is maintained. But without product detail pages, the store is currently a filtered gallery that leads nowhere. The store must complete Phase 3 (product detail template) before this principle can be fully honored.

Recommendations

Twenty specific recommendations, prioritized. Critical items are blockers for launch. High items weaken the site if unresolved. Medium items affect coherence. Low items are polish.

Critical — Pre-Launch Blockers

Strengths
  • Grid discipline is exemplary — consistent column assignments across all 7 pages and navigation create continuous key lines throughout the site
  • Color palette is correct and rigorously applied — 5 values, zero decorative use, nothing introduced for style

  • Art horizontal scroll — the site's formally most ambitious decision, using format constraint as compositional material

  • Reading list — the site's best content idea and strongest cultural voice moment
  • Mobile overlay navigation — bold and appropriate, full-screen with large display type

  • The brief is excellent — self-aware, specific, well-referenced, anti-generic
Weaknesses
  • 13 type styles is too many — the typographic architecture is diluted; 8–9 would make the remaining styles carry more weight and meaning
  • Home page name is undersized — the site's primary identity moment is cautious when it should be committed
  • No real photography — the site cannot communicate warmth, material specificity, or editorial atmosphere without it
  • Compositionally conservative — correct and clean but not yet bold; no moment makes a strong visual argument the way the cited references do
  • Eight spacing violations break the 4px system the brief explicitly mandates

The architecture is right. The vocabulary is right. The system mostly holds. What it needs now is commitment — bolder type, real images, and the willingness to push the grid to make an argument, not just an order.
