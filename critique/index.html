01

Overview

The foundations are correct. The ambition is genuine. The system mostly holds. What it is not yet is finished — and several current weaknesses will resolve with real content and a final pass. The ones that won't self-resolve are what this report is for.

This is a technically disciplined site built around a coherent editorial philosophy. Every page uses the same 12-column grid at the same gutter and gap values. The navigation participates in the grid at the same column widths as every content section. The three-color text hierarchy (text / muted / rule) is applied consistently throughout. The token system is comprehensive and mostly honored. The section differentiation — each page having a distinct character without becoming disconnected from the whole — is more difficult than it looks and the implementation earns it.

Where the site falls short: there are too many distinct type styles (13 is too many; 8 or 9 is the target). Some clamp() ranges don't land on type scale steps. A handful of spacing values violate the 4px base unit. Some viewport-relative values create positional unpredictability. And the site is compositionally conservative — correct and clean, but not yet bold. The layouts are technically right but don't yet make a strong visual argument in the way the best Swiss and editorial work does.

The biggest unresolved risk is not code — it is content. The site is designed around photography that does not yet exist. Until it does, no final judgment about atmospheric success is possible. The placeholder panels suggest the right scale and proportion relationships. But the editorial warmth the brief describes has no vehicle yet.

02

Methodology

Four frameworks applied simultaneously. No one frame is sufficient alone. Together they give a complete picture.

Heuristic Evaluation (Nielsen, 1994)

Jakob Nielsen's 10 usability heuristics were applied across all page templates. For this site, the most operative heuristics are: Consistency and standards (are spacing, type sizes, and interactive behaviors consistent?), Aesthetic and minimalist design (every element that is not load-bearing content is a liability — the principle most aligned with the Swiss aesthetic), and Visibility of system status (is location always clear within the site?). The audit notes where the site satisfies or violates each.

Swiss Design Principles (Crouwel, Hofmann, Müller-Brockmann)

Wim Crouwel: "Everything should be straightforward and informative — information was number one." His grid-based methodology for the Stedelijk Museum (1963–1985) established that a grid is not a constraint but a system of liberation. The grid should make the design possible, not just ordered. Application here: is the grid doing structural work, or just constraining placement?

Armin Hofmann: all visual communication reduces to point, line, and plane. His Graphic Design Manual (1965) presents visual contrast — black/white, large/small, static/dynamic — as the structural grammar of all graphic design. Application: evaluate each page for contrast ratios, scale drama, and figure/ground clarity.

Josef Müller-Brockmann: "The grid system is an aid, not a guarantee." It enables a range of possible solutions; the designer's judgment fills the space. Application: the grid alone does not produce good design. The question is what is done within it.

Elements of Typographic Style (Bringhurst)

Key principles applied: typography exists to honor content (all type choices serve the text, not the designer); the measure (45–75 characters per line; too wide breaks tracking, too narrow creates racing); hierarchy through scale and weight, not color; and the page is a proportion — the mathematics of the column and margin should be deliberate, not arbitrary.

Design Thinking / Intention Alignment

The brief package is treated as the primary evaluative standard. Design Thinking principles (empathy, problem definition, ideation, prototyping, testing) are applied to assess whether the site's design decisions are solving the right problems. The question is not only "is this well-executed?" but "is this the right problem to be solving?"

"I am a functionalist troubled by aesthetics." — Wim Crouwel. The self-description that should sit above this entire project.
03

Grid & Layout

The grid discipline is the strongest element of the site. Every page template uses the same system, and the column assignments are not just consistent — they are meaningful.

The 12-column grid with 24px column gap and 48px outer gutter is the structural backbone. Every section — including the navigation — uses repeat(12, 1fr) with identical padding. This creates continuous vertical key lines from top to bottom of every page: the wordmark, the nav links, the left content edge, the right rail, all share the same x-positions across all pages. This is Crouwel's principle in practice: the grid provides continuity across formats.

The consistent column assignment pattern — label/wordmark in cols 1–2, primary content in cols 3–9, right rail in cols 9–12 or 10–12 — mirrors the nav's own structure (wordmark 1–2, links 9–12). When the reader sees nav links and page content aligning to the same column, the system becomes legible. This is correct editorial thinking.

Column System
12-col, 24px gap
Consistent across all 7 pages and navigation. No exceptions at desktop.
Outer Gutter
48px desktop
Brief says "24px recommended default" — the implementation uses 48px for outer margin and 24px for col-gap. Ambiguous brief, defensible execution. 48px = 12× base unit.
Image Bleeds
Negative margin
Correctly implemented via margin-left: calc(-1 * var(--gutter)) and margin-right: calc(-1 * var(--gutter)). Bleeds are structural, not decorative.
Nav Integration
Grid-participant
Navigation uses the same 12-col grid and padding as every content section. Wordmark sits at cols 1–2, links at cols 9–12. This alignment with content keylines is exemplary.

Issues Found

Warning
Viewport-relative padding on Objects left panel
The left panel uses padding-top: 38vh to float the text in the lower half. Compositionally motivated, but at a 600px viewport height this equals 228px; at 1200px it equals 456px. This positional instability can produce unwanted results at unusual screen proportions. A fixed token (e.g. --sp-20: 160px) would be more predictable while achieving a similar effect.
objects.module.css → .left
Warning
Viewport-relative min-height on Image header
min-height: 40vh on the Image page header has the same unpredictability problem. At 600px viewport this is 240px of void; at 1400px it is 560px — which is close to the full viewport. A fixed extended spacing token would behave predictably.
image.module.css → .headerBody
Strength
Home page void — active negative space
Columns 5–8 of the homepage are intentionally empty. The comment in code reads "intentionally empty — negative space is part of the composition." This is the Hofmann principle of figure/ground in action. The void creates tension and pushes the text into the right third, where the nav links sit. This is compositionally correct and should not be changed.
home.module.css → comment at line 31
Strength
Art horizontal scroll — format as design
The works strip uses overflow-x: auto with card widths calculated from viewport height (calc(50svh * 6/7)). This treats the format constraint as a design element — the scroll is the content experience, not a workaround for overflow. This is the most formally ambitious layout decision on the site.
art.module.css → .works
Opportunity
Every page uses the same column logic — consider one bold exception
The grid discipline is correct and necessary. But Crouwel and Hofmann's best work creates visual tension within the grid, not just order. Currently no page makes a bold compositional argument of its own. One section — perhaps the Collection hero, or a future editorial spread — could use the grid more dramatically: full-bleed type at scale, an unexpected ratio, asymmetric content placement. The system is set up for this. The opportunity hasn't been taken yet.
04

Typography

The type system is correctly conceived but not yet fully rationalized. The scale is right. The weights are right. The line heights are right. Too many styles have been introduced, and some clamp() ranges escape the scale.

The Scale

The minor third (1.2) from a 14px base, with 4px-aligned fixed line heights, is correct and well-implemented in globals.css. The progression from 11.7px (xs) through 41.8px (H1) gives seven usable steps. Line heights from 16px to 48px are clean multiples that create genuine vertical rhythm. This is the right foundation.

Weight Convention

The three-weight system is applied correctly throughout most of the site: Light (300) for display and titles, Regular (400) for body text, Medium (500) for labels, tags, and metadata. This is the correct Swiss convention — weight does semantic work. One violation: the service card titles in Studio use Medium weight while functioning as display titles, not labels. This breaks the convention without justification.

Type Style Inventory

Across all 14 modules, the following distinct type treatments exist. Thirteen is too many. The brief calls for typography to function as architecture — architecture has few materials, used with precision. The target is 8–9 styles maximum.

The clamp() Problem

Six clamp() uses across the site set fluid type sizes. Four of these set min or max values that don't land on type scale steps. This means the fluid type jumps off the rational scale during viewport transitions. Bringhurst is explicit: the typeface's optical behavior at a given size matters — you can't slide arbitrarily between sizes. These ranges should be re-anchored to scale steps.

Violation
Home title: max value not on scale
clamp(20px, 2.5vw, 36px) — 36px is the line-height of H3, not a font size. H2 is 34.8px, H1 is 41.8px. The title of the site, the primary identity marker, currently sits between scale steps at its maximum. Given that the name is the only content on the home page, this deserves a committed size. Consider anchoring to H1 (2.985rem) as maximum.
home.module.css → .title
Violation
Collection hero title: both endpoints off-scale
clamp(32px, 5vw, 64px) — neither 32px nor 64px is in the type scale. This is the first text visitors see on a collection page, at full-bleed scale. It should be anchored: perhaps H1 (41.8px) to display (4.5rem = 63px), with a more precise minimum.
collection.module.css → .heroTitle
Warning
Objects entry title and Collection intro: off-scale minimums
clamp(18px, 2vw, 24px) and clamp(18px, 2vw, 26px) — 18px appears in both as a minimum, but 18px is between body (14px) and H6 (16.8px). These could use 16.8px (H6) as minimum and 24px (H4) as maximum for the first, and 16.8px to 28.8px (between H4 and H3) for the second.
objects.module.css → .entryTitle / collection.module.css → .intro
Opportunity
The home page name should be more committed
The brief says the site should not feel like a "Squarespace template minimalism." But the home page name "Andrew Whited" at a maximum of 36px in font-weight light reads cautiously. The Chipperfield site uses a typeface that carries the entire brand load. Andrew's name is the brand. At full viewport, it should be a presence, not a caption. Consider H1 (41.8px) or the display size (clamp up to 4.5rem) at font-weight light. The light weight is correct; the size is not.
05

Spacing

The spacing token system is well-designed and used correctly in most places. Eight violations exist — small in number, but the brief explicitly states "do not introduce arbitrary spacing values." These should be resolved.

The 4px base unit with named steps from xs (4px) through large-section (64px), and an extended scale to 160px, gives ample range. The structural use of 2px gaps in image grids and service card grids is a deliberate compositional choice (the "ruled gap"), not a system violation — this is correct design thinking. The problems are elsewhere.

Violation
10px padding on reading list rows
padding: 10px 0 on .readingLink — 10px is not a 4px multiple. Should be --sp-sm (8px) or --sp-md (12px). This affects every row in the reading list on Studio — approximately 12 instances.
studio.module.css → .readingLink
Violation
3px gaps in work cards and product info
gap: 3px in .workText (Art landing) and .productInfo (Store). 3px is not a 4px multiple. Should be --sp-xs (4px). This is a micro-detail but the build brief explicitly prohibits arbitrary values.
art.module.css → .workText / store.module.css → .productInfo
Violation
3px optical padding on label elements
padding-top: 3px appears in multiple label elements across Studio, Collection, and Work pages — a recurring optical alignment hack to vertically align uppercase labels with adjacent text. The intent is correct (baseline alignment), but the method is off-system. Either use 4px (--sp-xs) or solve it with a CSS translate or line-height adjustment.
studio.module.css, collection.module.css, work.module.css → .bioLabel, .workshopLabel, .metaLabel, etc.
Warning
min-height: 38px on reading list title
The .readingTitle has min-height: 38px — an arbitrary pixel value used to prevent text from clipping on short titles. 38px is close to 40px (--sp-xxxl), but not equal. Should use the token or remove the constraint and rely on line-height.
studio.module.css → .readingTitle
Warning
Store product grid top padding uses a fragile calc
padding-top: calc(var(--sp-12) + var(--lh-xl) + var(--sp-4)) — this complex expression attempts to align the product grid with the filter row by accounting for its line-height and spacing. The intention is correct (visual alignment), but the expression is brittle and depends on the filter label's exact rendering. If the filter changes, this breaks silently.
store.module.css → .grid
Strength
2px structural gaps — a deliberate compositional device
The 2px gaps between service cards, image grid cells, and art work images create a "ruled gap" that gives the compositions a printed-grid quality. These are not spacing violations — they are structural design decisions that produce a visual effect (the gap as rule, not as breathing room). This is correct.
studio.module.css, image.module.css, work.module.css
06

Color & Interaction

The color system is the strongest component of the site. Five values, rigorously applied. The warmth of the cream background creates the right atmospheric register without falling into lifestyle-brand territory.

The palette: background (#F6F6F2 cream), text (#111110 near-black), muted (#72726E gray), rule (#DEDED8 light border), placeholder (#3E3C38 dark). This is nearly monochromatic — the Swiss design principle of black, white, and one accent, applied to a warmer register. The cream versus near-black is a more considered pair than pure white on black: it reads as material and warm without softening the structure.

Palette Size
5 values
Minimum viable and sufficient. No color has been introduced for decoration. Müller-Brockmann would approve.
Hover Model
Opacity reduction
Consistent: 0.5 for nav/labels, 0.6–0.7 for cards. Opacity hover reads as refinement. 0.15s transition throughout.

Issues Found

Violation
Hardcoded rgba() values duplicate the background token
rgba(246, 246, 242, 0.5) and rgba(246, 246, 242, 0.3) appear in the lightbox meta and heroBack — these hardcode the background color value instead of referencing var(--color-bg). If the background color ever changes, these will drift. Should be implemented with CSS color-mix or a workaround like color: var(--color-bg); opacity: 0.5 on a wrapper.
image.module.css → .lightboxMeta, .lightboxCount / collection.module.css → .heroBack
Warning
Hardcoded placeholder variation in art work detail
The second image wrap uses background-color: #b0a89a — a specific warm gray that suggests a second tonal value for the placeholder system. This is intentional (differentiating the two image panels) but the color is not a system token and should be defined as one if it's intended as a permanent pattern.
work.module.css → .imageWrap:nth-child(2) .image
Opportunity
Corner-mark CTA: the site's one decorative gesture
The inquiry button in the art work detail uses 8 layered gradients to create growing corner marks on hover. This is the most complex interaction on the site and the only gesture that risks being ornamental rather than functional. The brief says "Restraint over decoration." This deserves a design conversation: is this a signature moment that earns its complexity, or is it the kind of "one-off gesture" the brief warns against? It's well-executed but its presence here, isolated from the rest of the site's simpler hover model, makes it feel imported from a different site.
work.module.css → .inquireBtn
Strength
Nav color states are thoughtfully managed
The three nav color states — .navLight (wordmark goes cream over dark image), .navLinksLight (links go cream over dark panel), .navOpen (overlay dark background) — are correctly implemented and cover the actual visual contexts of each page. This is functional foresight about how the fixed nav will interact with each section's image content.
Nav.module.css → .navLight, .navLinksLight, .navOpen

Page Analysis

Each page evaluated for design system adherence, compositional strength, brief alignment, and outstanding issues.

Home / A−

The clearest expression of the grid as composition. Image (cols 1–4), void (cols 5–8), text (cols 9–12). The void is active, not empty. The column assignment mirrors the nav, creating a top-to-bottom alignment at the right third of the page. This is correct Hofmann thinking.

Opportunity
The name is undersized for its purpose
The home page has one job: establish identity. Andrew Whited at clamp(20px, 2.5vw, 36px) in font-weight light is cautious for a name that carries the entire site. At 1440px this renders at approximately 36px. This is the H3 line-height, not the H1 size. The Chipperfield, Jasper Morrison, and Caruso St John sites all lead with the name at a committed, commanding size. The light weight is correct. The size should be larger — anchored to H1 (2.985rem) minimum.
Warning
Descriptor at xs size is barely legible at a glance
"Studio, objects, art, and image." at --text-xs (11.7px) with a max-width: 160px is the first and only text after the name. At 11.7px the line length will be approximately 20–25 characters — near the lower bound of Bringhurst's acceptable range (45–75). This descriptor is an important orienting statement. At xs size it reads as a caption, not a statement. Consider sm (14px) with wider max-width.
Strength
The void is the design
Cols 5–8 are intentionally empty and the code comment says so. This is a rare and correct decision — most designers fill every column. The void here creates breath, forces the eye to find the text, and creates a compositional tension. This should not be changed.
Studio /studio B+

The richest and most complex page. Eight sections — Hero, Bio, Workshop, Services, Reading, Listening, Contact. The reading list is the most distinctive content element on the site. The services section is the most generic. The contact section is unfinished.

Strength
Reading list — the site's best content idea
The "Required Reading" section — 12 items (books, films, essays) in a tabular list with floating hover thumbnails — establishes cultural voice without text-heavy biography. The structure (title, author, type, note) is rigorous. The floating thumbnails appearing on hover is a delightful micro-interaction that fits the reading experience. This is the page's high point.
Warning
Contact section is visually incomplete
The contact section uses grid-column: 1 / 7, leaving cols 7–12 empty. With no right-side content this reads as a half-finished layout — not as intentional negative space. Either add content (secondary contact, physical address, exhibition schedule) to the right, or redesign as full-width.
studio.module.css → .contactInner
Warning
Hero heading is very small for the opening statement
The studio hero heading uses clamp(15px, 1.5vw, 20px) — at 1440px this renders at 21.6px (between H6 and H5). The text in this position introduces the entire practice. At 20px it reads as body text, not as an editorial statement. Consider anchoring to H4 (1.728rem = 24px) or H3 (2.074rem = 29px) at maximum, with the light weight preserved.
Warning
Services section uses generic card layout
The four service cards — Custom Furniture, Spatial Installation, Material Consulting, Small-Batch Production — are presented in a 4-column card grid with 16/9 placeholder images and body text. This is the most "template" feeling section on the site. The brief warns against "startup portfolio" styling. The services section currently reads closest to one. Consider a different presentation: a list format, or a statement-led section, rather than icon-card grid.
Objects /objects A−

The split-panel layout with a sticky left panel and scrollable entry list is correctly conceived for an editorial discovery layer. The hover background image effect is the site's most distinctive interaction.

Strength
Hover background images — the site's most distinctive interaction
Clicking/hovering a collection row fades in a masked background image of the collection — centered, softened at edges with a linear gradient mask. This is image-led behavior: the work appears at the moment of interest, not preemptively. This is the Apparatus Studio approach: reveal work through interaction rather than displaying it statically. This should be developed and refined, not simplified away.
Warning
38vh push creates positional unpredictability
The sticky left panel uses padding-top: 38vh to float the title text in the lower third of the viewport. The intent — creating negative space above the text — is correct. But at a 500px viewport (some tablets in landscape) this is 190px; at 1200px it is 456px. Consider a fixed value (--sp-20: 160px or similar) that achieves similar spatial effect without viewport-dependency.
Opportunity
Entry typography could be stronger
The entry title at clamp(18px, 2vw, 24px) is reasonable but conservative. For a list that represents the primary editorial content of the Objects section, these titles could carry more weight — not in the CSS weight property, but in size. At 24px maximum they are H4 level. Consider H3 (29px) or H2 (35px) as maximum, letting the title be genuinely bold against the smaller descriptor text.
Art /art A−

The horizontal scroll works strip is the most formally ambitious layout on the site. The 50/50 viewport split, image cards at 50svh height, and the scrollable gallery format treat the browser as a display case. This is the right approach for artwork.

Strength
Horizontal scroll as format-conscious design
Card widths derived from calc(50svh * 6/7) maintain the 6:7 aspect ratio regardless of viewport height. Images are exactly 50svh — pinned to the viewport midpoint. This is not a workaround for overflow; it's a conscious treatment of the page as a horizontal gallery. Crouwel's principle: accept the format constraint and design within it.
Opportunity
The stage void needs to earn its space
The top half of the Art page is entirely empty except for a small label and intro text in cols 9–12. This is the largest void on any page. With placeholder content it reads as blank. With a strong artwork image or a compelling typographic moment in the void, it would read as powerful. The design is set up for something; it doesn't yet deliver it. Consider whether the stage void should do more work — perhaps showing a featured work, or using a large typographic element to fill the compositional weight.
Violation
0.01em tracking and 3px gap are off-system
letter-spacing: 0.01em on .workDetail is not a defined tracking token. The system has --tracking-wide: 0.06em and --tracking-wider: 0.1em. 0.01em is essentially no tracking; use 0 or use a token. The gap: 3px in .workText should be 4px.
Image /image A−

Clean and correctly designed. The 4-col grid with 2px gaps creates a precise contact-sheet quality appropriate for photography. The lightbox is well-engineered. The header correctly mirrors the Art page's right-pinned label convention.

Strength
Lightbox is correctly built
The visibility+opacity transition pattern, keyboard navigation (Escape, arrow keys), the 56px bar with set metadata — location, year, counter — and the clean close button are all correctly implemented. The lightbox stays within its photo set scope. This is functional and restrained.
Warning
6:7 aspect ratio will crop landscape photography
All grid cells use aspect-ratio: 6/7 — portrait. If the photography includes significant landscape-oriented images, these will be cropped. Consider whether a 1:1 grid or mixed ratios might serve the actual photography better, or confirm that all image work is portrait-dominant.
Store /store B+

The store handles functional complexity — filtering, state management, empty states — without compromising the editorial aesthetic. The left-panel filter / right-panel grid structure correctly mirrors Objects, creating continuity between the discovery and transaction layers.

Violation
Filter label weight breaks the label convention
.filterLabel uses font-weight: var(--font-light) — the only label in the system using light weight instead of medium. All other labels (section headers, metadata, tags) use medium weight. This creates an inconsistency in the filter UI that reads as unfinished.
store.module.css → .filterLabel
Warning
No product detail page template visible in codebase
The Store landing and product grid are complete, but no /store/[slug] page template exists in the visible file structure. The build brief places Product Detail in Phase 3 ("after editorial system feels correct"). Given the Store is functionally in place, the detail page is now the critical gap for Phase 3 completion.
Strength
Filter active state indicator is well-designed
The ::after dot indicator on active filter buttons uses 12px circular filled dot, background-color: currentColor. It's clear, small, and non-decorative. This is the correct approach for an active state — signal without noise.
Navigation Components/Nav A

The navigation is one of the site's strongest design decisions. It participates in the 12-column grid at the exact column positions as page content, creating continuous key lines. The mobile overlay with large display type is bold and correct.

Strength
Grid-integrated navigation
Wordmark at cols 1–2, links at cols 9–12. This is identical to the content column assignments on every page. The vertical key lines created by the grid extend from nav through every section. This is what Crouwel's Stedelijk system did: consistent alignment across all formats. This decision is architecturally correct and should be preserved absolutely.
Strength
Mobile overlay — bold and appropriate
The full-screen dark overlay with large display type links is a strong contrast to the site's otherwise small type. The bottom-alignment, large tracking, and border separators make this the most typographically dramatic moment on mobile. This is correct: mobile navigation should command the full screen rather than hiding in a tiny drawer.
Warning
Overlay link clamp endpoints off-scale
clamp(32px, 9vw, 56px) for overlay links — neither 32px nor 56px is a type scale step. At 9vw on a 375px phone this is 33.75px ≈ H2 territory, which is fine. But the anchoring should reference the scale. H2 (34.8px) to a display value (56px ≈ --text-display max) would be cleaner.

Market & Inspiration

The design philosophy is correctly oriented relative to the competitive field. The influences named in the brief are legible in the work. The site is not there yet, but the vector is right.

Competitive Landscape

The market for this site sits at the intersection of boutique furniture, art practice, and editorial photography — a specific intersection that is not well-served by most portfolio platforms. The closest parallels are studios that operate at the gallery-design boundary: Apparatus Studio, Muller Van Severen, Rick Owens furniture (represented through Salon 94 Design and Carpenters Workshop Gallery), and design-literate architects like Caruso St John and 6a.

Reference What They Do Well Lesson for This Site
Apparatus Studio Site IS the editorial voice. Atmospheric photography, restrained type, gallery-not-store feeling. Commerce present but not dominant. The site should feel like an authored object, not a display platform. Currently correct in structure; needs photography to fulfill it.
Jasper Morrison "Super Normal" philosophy. The UI disappears; the object is the interface. Typology-organized, store integrated without commerce dominating. Most directly relevant comparison. Morrison's site is less complex in structure (one practice type) but the same restraint principle applies.
Muller Van Severen Extreme restraint. Photographs of pieces in natural light. Almost no copy. Flat list navigation. The work looks deceptively simple — the site mirrors it. When the design is stripped down, material quality and form carry everything. Confirms the palette and typographic restraint as correct.
David Chipperfield Custom typeface carries entire brand load. White on white on white. Identity through absence. "Simple, Ordinary, Complex" — three words do the positioning. Invest in the typeface. Aktiv Grotesk is a good choice. Use it with enough conviction that the type itself carries identity.
Caruso St John Text-led at times, image-led at others. Restrained site mirrors restrained practice. Strong typographic hierarchy. The reading list section on Studio is already doing this — cultural references doing character work. Develop this approach.

Crouwel and Hofmann — Alignment Assessment

The site correctly reflects several principles from both designers. Grid discipline, spare palette, sans-serif type, structural use of negative space. The Art horizontal scroll is the closest thing to a Crouwel-level compositional argument: it accepts the browser format as a constraint and designs within it, producing something that could not exist in print.

Where the site doesn't yet reach the level of the references: neither Crouwel nor Hofmann produced work that was merely correct. Their work creates visual tension — dramatic scale contrast, strong figure/ground relationships, compositional arguments. The site is ordered and disciplined but not yet tense. There is no moment of the Hofmann theater poster: large type at a scale that commands, against an image that pushes back.

"The grid system is an aid, not a guarantee. It enables a number of possible uses and each designer can look for a solution appropriate to his personal style. But one must learn how to use the grid; it is an art that requires practice." — Josef Müller-Brockmann, Grid Systems in Graphic Design

Where the Site Can Push

Opportunity
Scale contrast — the missing dramatic element
The type sizes are modest and consistent across the site. The largest settled type on any page is approximately 24–29px. The largest display type (collection hero) reaches 64px but only through clamp. No section makes a bold typographic statement at large scale. Consider one section where type is the composition — a full-width statement heading at H1 or display size, perhaps on the Studio hero or the Objects left panel, that creates genuine scale contrast against the small body type below it.
Opportunity
Type lockups — there are none yet
A type lockup is a designed typographic composition — name + discipline + location, or a statement set with intentional visual relationships between size, weight, and spacing. The home page has the elements for a lockup (name + descriptor) but doesn't commit to the composition. The Studio hero has the opportunity. The best editorial sites have at least one moment where the type is a designed object, not just a label. This doesn't exist yet.
Opportunity
The Collection pages are the closest to editorial ambition — develop them
The Collection detail pages — with 70svh full-bleed hero, overlaid title, intro text, labeled sections, references — are the closest the site gets to the editorial catalog mode that distinguishes the best boutique furniture presentation. This template deserves the most photographic and copywriting investment. When a collection has great photography and precise text, this page format will deliver.

Intention vs. Execution

The brief is unusually good. It is specific, self-aware, and anti-generic. Most of it is being honored. A few principles are being honored in structure but not yet in practice.

What the Brief Says — What the Site Does

Strength
"Editorial over corporate" — Being honored
The layouts do not read as templates. The Objects hover effect, the Art horizontal scroll, the Collection hero — these feel authored. The site does not look like Squarespace.
Strength
"Restraint over decoration" — Being honored
There is one exception (corner-mark CTA) but the site is otherwise clean. No gradients, no shadows, no illustration, no icon libraries. The restraint is genuine.
Strength
"Distinct modes within one system" — Being honored
Home (hero split), Studio (long-form), Objects (list), Art (scroll gallery), Image (grid), Store (filter+grid) — each is visually distinct while sharing the same type, spacing, color, and grid vocabulary. This is the site's strongest achievement.
Warning
"Typography as architecture" — Partially honored
The grid uses type correctly as structure. But with 13 type styles, the typographic architecture is not as clear as it should be. Architecture has few materials, used with precision and intention. When the type system is tightened to 8–9 styles, the remaining styles will carry more weight and meaning.
Warning
"Image-led experience" — Structurally honored, atmospherically pending
Every layout prioritizes image in scale and placement. The bleeds, the horizontal scroll, the 70svh collection hero — these are image-led decisions. But there are no images. The atmosphere the brief describes ("warmth should come from the work itself") cannot be assessed until photography exists. This is the site's most significant open question. All placeholder panels should be replaced with actual work as early as possible.
Warning
"Negative space should be active and meaningful" — Mostly honored, one exception
The home void is active. The Art stage void is motivated. But the Objects left panel's 38vh push creates negative space above the text that feels positional rather than compositional. Active negative space creates tension; positional emptiness is just a gap. The 38vh value should be reconsidered as a compositional decision, not a layout trick.
Warning
"Commerce without dilution" — Structurally correct, functionally incomplete
The Store doesn't feel like Amazon. The editorial hierarchy is maintained. But without product detail pages, the store is currently a filtered gallery that leads nowhere. The store must complete Phase 3 (product detail template) before this principle can be fully honored.

Verdict

B+. A disciplined, intelligent, system-driven site that is not yet finished — and whose strongest qualities are structural rather than atmospheric. The system is correct. The ambition is present. The voice is not yet audible.

Strengths
  • Grid discipline is exemplary — consistent column assignments across all 7 pages and navigation create continuous key lines throughout the site
  • Color palette is correct and rigorously applied — 5 values, zero decorative use, nothing introduced for style
  • Section differentiation without disconnection — each page has a distinct mode while sharing the same typographic and spatial vocabulary
  • Art horizontal scroll — the site's formally most ambitious decision, using format constraint as compositional material
  • Objects hover background effect — the most distinctive interaction, image-led and editorial in character
  • Navigation participates in the grid — one of the most disciplined structural decisions in the codebase
  • Reading list — the site's best content idea and strongest cultural voice moment
  • Mobile overlay navigation — bold and appropriate, full-screen with large display type
  • Token system is comprehensive — the brief's "no arbitrary values" standard is mostly met
  • The brief is excellent — self-aware, specific, well-referenced, anti-generic
Weaknesses
  • 13 type styles is too many — the typographic architecture is diluted; 8–9 would make the remaining styles carry more weight and meaning
  • Home page name is undersized — the site's primary identity moment is cautious when it should be committed
  • No real photography — the site cannot communicate warmth, material specificity, or editorial atmosphere without it; this is not a code problem but it is the largest risk
  • Compositionally conservative — correct and clean but not yet bold; no moment makes a strong visual argument the way the cited references do
  • Eight spacing violations break the 4px system the brief explicitly mandates
  • clamp() ranges escape the type scale at key moments
  • Services section reads as generic card grid — closest thing on the site to "startup portfolio"
  • Product detail page missing — Phase 3 commerce layer is structurally incomplete
  • Contact section unfinished — empty right half
  • No type lockup anywhere — no moment where type is a designed composition, not just a label
B+
The architecture is right. The vocabulary is right. The system mostly holds. What it needs now is commitment — bolder type, real images, and the willingness to push the grid to make an argument, not just an order.

This is a strong foundation for a genuinely distinctive site at the intersection of object design, art practice, and editorial photography. The brief describes something specific and rare: "editorial, not corporate — refined, not cliché-luxury — minimal, but not empty." That is exactly what the site could be. It is not yet there. But the structure that would support it is almost entirely in place. The remaining work is less construction than it is finishing — and a few bold design decisions that are currently being left on the table.
